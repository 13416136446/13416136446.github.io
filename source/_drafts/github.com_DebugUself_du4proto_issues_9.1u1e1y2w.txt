## 版本仓库要放什么?
- 项目中所有的东西都应该放在仓库中
- 什么是项目需要的?
- 如果没有这个文件, 软件工程就无法生成/交付和运行
- 自动生成的文件要格外注意
    + 格式
    + 产生时间
- github 
    - .gitignore
        + 40种配置
        + python 的
        + 每种开发语言相关编辑器等都会有辅助/无用的文件生成
        + github 收集完备
    - 开分支
        + 将所有文件删光
        + 包括. gitignore
    - 无关文件
        + meta (源数据信息文件)
        + 与项目无关
        + 工作物件涵盖了所有项目文件,我们都要处理.
            + 分散了同伴精力
            + 泄露隐私
- 放什么文件反映对当前项目的理解
    - 你是否知道你在做什么?
    - 图片这些是不应该出现的
    - 项目说明文档/框架图/ email 文本这些应该放
    - 版本仓库是保鲜库, 恢复项目环境
- 图片
    + 课程中的 demo 网站中的图片
        + 静态的, 美化用, 对工程无用.
        + 照片太大, 占用流量和内存
- 小结
    - 认真看待这个问题
    - 了解你的项目
    - 了解那些不应该放进去的文件和后置文件格式, 是什么
    - 有这样的意识之后, 规范行为.    

- 延伸(读文档)
    + 版本仓库应该放什么的内容
        * 源代码
        * 其他生成代码的应用程序
        * 源数据文件
        * 项目文件/项目文档
    + 不该放的内容
        * 可被容易再次生成的文件不需要放
        * 自动生成的文件
- 行动
    + 尝试创建分支时, 保留.gitignore
    + 尝试了解.gitignore 里面的文件及格式是什么
    + 根据自己项目的需要, 修改 .gitignore
    + 描述自己的项目应该出现哪些格式的文件
